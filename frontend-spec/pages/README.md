# Demo Page Registry

本文件用于登记所有 Demo Page，避免页面存在但无人维护。

## 命名规则

- 文件统一放在 `frontend-spec/pages/`
- 文件名格式：`task-<两位任务编号>-<page-slug>.page.md`
- 一个 Task 对应多个页面时，继续细分 slug，不新增子目录

## 维护规则

1. 新增页面文档时，先更新本表。
2. 页面状态必须和实际开发状态同步。
3. 如果一个 Task 对应多个页面，本表按页面拆行登记。
