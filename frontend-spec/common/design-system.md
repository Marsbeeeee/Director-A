# Design System

本文件定义 Demo Page 的统一视觉和布局基线，避免每个页面各自起一套样式。

## 1. 页面基线

- 设计目标：像内容页面，不像后台页，也不像简单 PPT。
- 默认场景：横屏演示优先，同时兼容手机竖屏。
- 页面层级：背景层 -> 内容层 -> 交互层 -> 状态层。

## 2. 布局规则

- 页面容器使用单一主内容列，不做多栏后台布局。
- 桌面端主内容宽度建议控制在 `1200px` 内。
- 移动端左右安全边距不低于 `16px`。
- 首屏必须在不滚动的情况下看到标题、核心剧情和主 CTA。

## 3. 视觉 Token

建议统一使用 CSS 变量，至少包含以下语义层：

```css
:root {
  --bg-page: #0d1020;
  --bg-panel: rgba(10, 14, 28, 0.72);
  --bg-overlay: rgba(6, 8, 16, 0.56);
  --text-primary: #f7f8fb;
  --text-secondary: #c6cad6;
  --text-muted: #8e96a8;
  --accent-primary: #ff6b57;
  --accent-secondary: #ffd166;
  --border-soft: rgba(255, 255, 255, 0.12);
  --state-success: #3ddc97;
  --state-warning: #ffb020;
  --state-error: #ff5d73;
}
```

要求：

- `accent` 只用来表达关键选择和高优先级动作。
- `error`、`warning`、`success` 不与主 CTA 颜色混用。
- 背景允许使用渐变、虚化、视频封面，但正文对比度必须达标。

## 4. 字体与层级

- 标题区：用于建立节目感和剧情张力，字号需要明显大于正文。
- 剧情摘要：控制在 2 到 4 行，不能写成长段说明。
- 操作按钮：按钮文案使用动词开头，如“开始观看”“进入分支”“重试加载”。
- 辅助信息：状态说明、时间戳、来源标签统一使用次级文本颜色。

## 5. 区块规则

每个 Demo Page 至少应包含以下三类区块：

- 内容区：承载剧情、角色、视频或图像主体。
- 理解区：帮助用户在 3 秒内看懂当前处于什么剧情节点。
- 动作区：给出下一步行为，不让用户停在无动作状态。

## 6. 响应式要求

- 横屏时，背景画面可占更大视觉比重；信息区可以悬浮覆盖。
- 竖屏时，区块改为垂直堆叠，CTA 固定在可视区域下部。
- 视频、封面、剧情卡片都必须支持窄屏裁切，不允许文字被遮挡。

## 7. 动效要求

- 动效只服务于“进入剧情”和“状态反馈”，不做装饰性堆砌。
- 页面进入动效建议控制在 `200ms` 到 `500ms`。
- CTA 点击后必须有明确反馈：高亮、loading、进度提示三者至少占一个。

## 8. 禁止项

- 不使用默认浏览器蓝色链接作为主要交互样式。
- 不让页面首屏只剩一张图和一个按钮，必须同时给出剧情上下文。
- 不把错误态写成技术报错信息，如 `request failed` 或 `500 error`。
