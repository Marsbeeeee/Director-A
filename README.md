# 导演A（Director A）

**AI for Any Audience · Action**
一个让观众参与决策的互动广告系统

导演A（Director A）是一个以 AI 为驱动的互动广告系统。
它通过将 Audience 引入叙事决策，让广告从“单向传递”转变为“参与式体验”。

在这个系统中：
- Audience 不再只是观看者  
- AI 不只是生成工具  
- 广告不再是打断，而是剧情的一部分  

而“Action”，来自用户每一次选择。

## 项目说明

### 产品定位

导演A解决的是一个核心问题：

> **广告是单向叙事，而用户注意力是双向选择。**

当前长视频平台广告存在：

1. 用户被动接受，无法参与
2. 广告干扰体验，甚至会员也无法完全规避
3. 广告内容与用户兴趣脱节，转化效率低

导演A通过“互动叙事”重构广告逻辑：

* 观众在关键节点做选择
* AI 实时生成不同剧情分支
* 广告内容嵌入剧情，而非打断内容

---

### 核心价值

* 将广告从“打断体验”变为“参与体验”
* 提升用户停留时间与互动意愿
* 提供更高质量的用户意图数据（选择行为）
* 为品牌提供可分支叙事能力，而非单一广告素材

---

## 核心功能

* **互动剧情分支**

  * 在关键节点提供选择（A/B/多选）
  * 不同选择进入不同广告叙事路径

* **AI 实时生成**

  * 基于用户选择动态生成剧情
  * 支持不同品牌内容嵌入

* **用户行为建模**

  * 记录选择路径
  * 构建用户偏好画像

* **广告融合机制**

  * 品牌内容作为剧情一部分
  * 避免传统“插播式”打断

* **多终端适配**

  * 长视频平台
  * 移动端短视频
  * H5 互动广告

---

## 技术栈

* **前端**：React / Web Interaction Engine
* **后端**：Node.js / FastAPI（可选）
* **AI**：LLM（用于剧情生成与分支逻辑）
* **数据**：用户行为数据 + 选择路径分析
* **可视化**：剧情树（Decision Tree）结构

---

## 项目结构


## 使用方式

### 典型用户流程

1. 用户进入视频内容

2. 广告剧情开始

3. 在关键节点出现选择：

   例：

   * “继续追剧情”
   * “查看产品信息”

4. 用户选择后进入不同剧情分支

5. AI 根据路径动态生成后续内容

6. 系统记录用户行为数据
