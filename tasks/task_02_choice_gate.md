# Task 02: 礼物选择互动节点

- Status: Planned
- Owner: Product / Frontend / Backend
- Last Updated: 2026-04-22
- Source Of Truth: 本文档为 Task 02 当前交付说明
- Depends On: `tasks/task_01_story_entry.md`
- Related Docs: `demo_flow.md`, `TEMPLATE_task.md`

## 目标

把恋综片段中的“礼物选择”落成一个真实可点击的互动节点，让观众明确感知自己在替男嘉宾做决定。这个任务要解决“广告如何变成剧情决策点”。

## 本期范围

1. 在剧情推进到关键节点时弹出互动卡片。
2. 提供两个明确选项：`送香水`、`送口红`。
3. 用户点击后，系统立即记录选择并进入对应分支。
4. 节点文案需要说明这是第一次见面的礼物选择，不是商品列表页。

## 非目标

1. 本期不做多选题和开放输入。
2. 本期不做真实电商跳转。
3. 本期不引入第三个礼物选项。

## 关键场景

1. 场景一：用户看到互动卡片，并能在 1 次点击内完成选择。
2. 场景二：用户未立即选择时，剧情暂停或停留在节点，避免直接跳过。
3. 场景三：用户完成选择后，系统进入对应剧情分支，并记录本次决策。

## 输入 / 输出

### 输入

- 用户输入：点击 `送香水` 或 `送口红`。
- 上游模块输入：当前剧情节点 ID、当前角色信息。
- 必填字段：`node_id`、`option_id`、`option_label`。
- 可选字段：节点说明文案、默认停留时长。

### 输出

- 前端展示输出：互动选择卡片、点击反馈、分支跳转。
- 后端接口输出：选择结果事件，至少包含 `node_id` 和 `option_id`。
- 持久化输出：写入一次 `choice_made` 行为记录。

## 状态与异常

- 成功态：点击后高亮选项，并进入对应分支。
- 低置信态：接口响应慢时，页面需有“正在进入剧情”反馈。
- 空结果态：节点配置缺失时，显示默认错误提示，不允许展示空按钮。
- 失败态：提交失败时允许重试，不丢失已选项。
- 权限或准入限制：默认无权限限制。

## 数据 / 接口 / UI 约束

- 依赖的数据契约：选择节点必须包含 `question`、`options[]`、`next_branch_id`。
- 涉及的 API：可新增 `POST /interaction/choice`。
- 是否允许改 schema：允许新增互动节点配置结构。
- 是否允许改接口：允许新增选择提交流程。
- 是否要求兼容现有前端行为：不能破坏普通剧情播放流程。
- 是否需要移动端适配：需要，按钮在手机上必须能单手点击。

## 前端规范补充

- 页面归属：剧情播放器内的互动层。
- UI 变更点：增加互动卡片、选项按钮、提交反馈状态。
- 前端消费规则：分支 ID 由后端或配置返回，前端不能自己硬编码剧情跳转关系。
- 页面文档同步：若存在播放器页面文档，应补充互动节点说明。

## 落地范围

- 需要修改的模块：播放器交互层、节点配置、选择事件处理。
- 可以新增的文件类型：节点配置、接口 mock、交互态样式。
- 本期不应改动的模块：AI 文案生成、行为分析报表页。
- 测试要求：验证点击、重复点击、弱网和移动端触控。

## 验收

1. 剧情推进到节点时，页面必须出现两个明确选项：`送香水`、`送口红`。
2. 用户点击任一选项后，页面立即给出反馈，并进入对应剧情。
3. 系统能记录用户选了哪个礼物，后续可供分支和行为分析使用。
4. 节点加载失败时，页面不能白屏，必须给出可理解提示。

## 验收示例

1. 当用户点击 `送香水` 时，系统记录 `option_id=perfume` 并进入香水分支。
2. 当用户点击 `送口红` 时，系统记录 `option_id=lipstick` 并进入口红分支。
3. 当接口 3 秒内未返回时，页面显示“正在进入剧情”而不是卡死。
