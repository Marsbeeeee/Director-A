# Task 01: 恋综主剧情入口页

- Status: Planned
- Owner: Product / Frontend
- Last Updated: 2026-04-22
- Source Of Truth: 本文档为 Task 01 当前交付说明
- Depends On:
- Related Docs: `demo_flow.md`, `TEMPLATE_task.md`, `frontend-spec/pages/task-01-story-entry.page.md`

## 目标

落地一个可直接进入 Demo 的恋综开场片段，让观众先进入剧情，再进入互动节点。这个任务要解决“用户一进来看到的到底是什么”这个问题。

## 本期范围

1. 提供一个可打开的 Demo 首屏，默认进入恋综内容而不是品牌介绍页。
2. 首屏明确展示恋综背景信息：男嘉宾、第一次正式约会、正在准备礼物。
3. 在主画面或字幕中抛出问题：`第一次正式见面，他该带什么礼物？`
4. 保证从首屏进入后，用户能自然过渡到后续选择节点。

## 非目标

1. 本期不实现真实视频素材剪辑系统。
2. 本期不要求接入真实品牌商品信息。
3. 本期不处理多节目、多场景切换。

## 关键场景

1. 场景一：评审首次打开 Demo，系统直接进入恋综剧情首页，并看到约会礼物问题。
2. 场景二：如果素材未加载完成，系统至少显示静态封面、剧情背景和继续进入按钮。
3. 场景三：用户点击“开始观看”或自动播放结束后，能进入 `Task 02` 的互动选择节点。

## 输入 / 输出

### 输入

- 用户输入：点击进入 Demo 或开始播放。
- 上游模块输入：恋综场景标题、背景文案、封面图或视频片段。
- 必填字段：节目标题、剧情简介、问题文案。
- 可选字段：旁白文本、字幕、倒计时或自动播放时长。

### 输出

- 前端展示输出：恋综主剧情入口页。
- 后端接口输出：本期可先无独立接口，允许本地静态配置。
- 持久化输出：可记录一次 `demo_entry` 事件。

## 状态与异常

- 成功态：用户看到恋综片段并理解当前剧情前提。
- 低置信态：只有封面图没有视频时，仍能读懂剧情问题。
- 空结果态：素材缺失时，显示占位封面和文案，不允许白屏。
- 失败态：页面加载失败时，提供重试入口。
- 权限或准入限制：无登录要求，默认所有演示用户可进入。

## 数据 / 接口 / UI 约束

- 依赖的数据契约：入口页至少要有 `title`、`scene_summary`、`question_text`。
- 涉及的 API：本期可不依赖线上接口，允许前端静态数据。
- 是否允许改 schema：允许新增 Demo 场景配置结构。
- 是否允许改接口：允许后续补充读取场景配置接口。
- 是否要求兼容现有前端行为：若已有首页，需保留原始进入方式。
- 是否需要移动端适配：需要，首屏信息在手机上不能折叠丢失。

## 前端规范补充

- 页面归属：Demo 首页 / 首个剧情画面。
- UI 变更点：增加恋综标题区、剧情说明区、问题文案区、进入动作区。
- 前端消费规则：前端不能自己猜剧情问题，必须由配置明确提供。
- 页面文档同步：对应页面文档为 `frontend-spec/pages/task-01-story-entry.page.md`。

## 落地范围

- 需要修改的模块：Demo 首页、场景配置、基础播放容器。
- 可以新增的文件类型：场景配置文件、占位素材、页面文档。
- 本期不应改动的模块：AI 生成逻辑、行为分析模块。
- 测试要求：验证桌面端和移动端都能正确显示剧情入口。

## 验收

1. 用户打开 Demo 后，第一眼看到的是恋综剧情，不是品牌页。
2. 页面上能明确读到“第一次正式见面，他该带什么礼物？”这个问题。
3. 即使没有真实视频，也能通过封面和文案说明清楚场景。
4. 用户能从当前页顺利进入互动选择节点。

## 验收示例

1. 当评审打开 Demo 时，页面显示恋综标题、男嘉宾准备约会的场景描述和礼物问题。
2. 当视频素材未加载成功时，页面仍显示静态封面和“进入下一步”按钮。
3. 当用户点击进入后，系统跳转或推进到礼物选择节点，而不是停留在首页。
