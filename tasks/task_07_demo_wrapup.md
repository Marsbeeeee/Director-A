# Task 07: Demo 收束页与讲解链路

- Status: Planned
- Owner: Product / Frontend
- Last Updated: 2026-04-22
- Source Of Truth: 本文档为 Task 07 当前交付说明
- Depends On: `tasks/task_01_story_entry.md`, `tasks/task_02_choice_gate.md`, `tasks/task_03_perfume_branch.md`, `tasks/task_04_ai_generation.md`, `tasks/task_05_lipstick_branch.md`, `tasks/task_06_behavior_panel.md`
- Related Docs: `demo_flow.md`, `TEMPLATE_task.md`

## 目标

把 Demo 最后一页或最后一段话术做成可直接收束的结果页，帮助评审在最短时间内记住 Director A 的核心价值链路。

## 本期范围

1. 输出一页清晰的 Demo 总结页，或在最终页面上展示总结信息区。
2. 页面要能复述完整链路：`恋综剧情 -> 用户做选择 -> AI 生成分支 -> 品牌融入剧情 -> 沉淀行为数据`。
3. 提供一版适合现场 30 秒讲完的总结文案。
4. 页面需要和前面步骤形成闭环，而不是另起一套术语。

## 非目标

1. 本期不输出完整商业计划书。
2. 本期不做品牌销售方案页。
3. 本期不做多行业扩展场景说明。

## 关键场景

1. 场景一：评审看完剧情和数据页后，进入总结页，能一眼读懂 Demo 的价值。
2. 场景二：如果没有独立总结页，也要在结果区呈现这条链路。
3. 场景三：现场时间不够时，讲解者只念一句话版本，也能收束全场。

## 输入 / 输出

### 输入

- 用户输入：点击“查看总结”或自动进入总结页。
- 上游模块输入：前面步骤的结果摘要、行为数据摘要。
- 必填字段：价值链路文案、简版总结文案。
- 可选字段：当前路径摘要、分支对比摘要。

### 输出

- 前端展示输出：总结页、链路图、30 秒讲解文案。
- 后端接口输出：本期可无独立接口，允许静态配置。
- 持久化输出：可记录一次 `demo_summary_viewed` 事件。

## 状态与异常

- 成功态：总结链路完整、文案统一。
- 低置信态：若前序数据不全，仍展示固定版价值链路。
- 空结果态：无前序摘要时，展示标准总结话术。
- 失败态：总结页加载失败时，至少保留一句话版本。
- 权限或准入限制：无额外限制。

## 数据 / 接口 / UI 约束

- 依赖的数据契约：总结页至少需要 `core_chain`、`short_pitch`。
- 涉及的 API：本期可无独立接口。
- 是否允许改 schema：允许新增 Demo 总结配置字段。
- 是否允许改接口：允许后续新增结果摘要接口。
- 是否要求兼容现有前端行为：如无总结页，可附加在结果页底部。
- 是否需要移动端适配：需要，链路文案不能因换行失真。

## 前端规范补充

- 页面归属：Demo 结果尾页 / 总结区。
- UI 变更点：增加价值链路展示区、短讲稿展示区、关键点摘要区。
- 前端消费规则：总结页使用统一文案源，不允许不同页面各写一版说法。
- 页面文档同步：如有 Demo 页面文档，应补充总结页说明。

## 落地范围

- 需要修改的模块：结果页、总结区、文案配置。
- 可以新增的文件类型：总结页文案配置、链路图说明文件。
- 本期不应改动的模块：互动节点逻辑、AI 生成接口。
- 测试要求：验证桌面端、移动端和无前序数据场景。

## 验收

1. 页面必须出现完整链路：`恋综剧情 -> 用户做选择 -> AI 生成分支 -> 品牌融入剧情 -> 沉淀行为数据`。
2. 页面必须提供一版 30 秒内能讲完的总结文案。
3. 即使前面某些数据未准备好，也能完成收束。
4. 评审看完这一页后，能复述 Demo 的核心价值。

## 验收示例

1. 当评审进入总结页时，页面显示完整价值链路和一句话版本。
2. 当前序行为数据为空时，总结页仍显示标准价值链路，不出现占位乱码。
3. 当讲解者时间不足时，只念“一句话版本”也能完成收束。
