# Task 06: 行为数据与路径可视化

- Status: Planned
- Owner: Product / Frontend / Backend
- Last Updated: 2026-04-22
- Source Of Truth: 本文档为 Task 06 当前交付说明
- Depends On: `tasks/task_02_choice_gate.md`, `tasks/task_03_perfume_branch.md`, `tasks/task_05_lipstick_branch.md`
- Related Docs: `demo_flow.md`, `TEMPLATE_task.md`

## 目标

把“用户做过什么选择、进入了哪条路径”展示成一页可讲的行为数据面板，证明 Director A 不只有曝光数据，还有剧情里的意图数据。

## 本期范围

1. 展示用户当前选的是香水还是口红。
2. 展示用户进入了哪条剧情路径。
3. 展示用户在路径中的停留或完成情况。
4. 提供一个简单的路径可视化或选择日志列表，支持现场讲解。

## 非目标

1. 本期不做完整 BI 系统。
2. 本期不做跨用户聚合报表。
3. 本期不做长期画像算法。

## 关键场景

1. 场景一：Demo 演示完两个分支后，切到数据页，能立刻看到不同选择路径。
2. 场景二：如果当前只有一条路径数据，页面也能解释清楚用户做了什么选择。
3. 场景三：如果行为日志获取失败，页面仍可显示本地最近一次选择结果。

## 输入 / 输出

### 输入

- 用户输入：可点击查看路径详情或切换数据视图。
- 上游模块输入：选择日志、分支进入日志、停留时间或播放完成度。
- 必填字段：`selected_option`、`branch_id`、`timestamp`。
- 可选字段：`dwell_time`、`profile_tag`、`completion_status`。

### 输出

- 前端展示输出：行为日志列表、路径标签、简单路径图或摘要卡片。
- 后端接口输出：可新增 `GET /user/behavior-path`。
- 持久化输出：本期读取已有行为事件，不额外新增写入动作。

## 状态与异常

- 成功态：用户能看到路径和选择记录。
- 低置信态：停留时间缺失时，页面只展示路径与选择结果。
- 空结果态：无日志时，页面展示“尚无行为数据”及说明。
- 失败态：接口失败时，展示最近一次本地记录或错误提示。
- 权限或准入限制：仅演示用户可查看本次会话数据。

## 数据 / 接口 / UI 约束

- 依赖的数据契约：至少有 `event_type`、`option_id`、`branch_id`、`timestamp`。
- 涉及的 API：`GET /user/behavior-path` 或本地 mock。
- 是否允许改 schema：允许新增前端消费用的聚合字段。
- 是否允许改接口：允许新增行为路径读取接口。
- 是否要求兼容现有前端行为：若暂无数据页，可独立新增 Demo 数据页。
- 是否需要移动端适配：需要，但可简化为卡片列表。

## 前端规范补充

- 页面归属：Demo 结果页 / 数据页。
- UI 变更点：新增路径摘要区、行为日志区、标签区、空态区。
- 前端消费规则：前端不能假设所有事件都存在，要按字段有无做兼容展示。
- 页面文档同步：如有数据页文档，应新增 Demo 路径页说明。

## 落地范围

- 需要修改的模块：行为事件读取、结果页 UI、路径展示组件。
- 可以新增的文件类型：mock 行为数据、路径图配置、页面文档。
- 本期不应改动的模块：AI 生成逻辑、剧情节点交互逻辑。
- 测试要求：验证有数据、无数据、接口失败三种状态。

## 验收

1. 页面必须能明确显示用户选了香水还是口红。
2. 页面必须能明确显示用户进入了哪条剧情路径。
3. 即使数据不完整，页面也能讲清最核心的用户选择结果。
4. 评审能通过这个页面理解“Director A 记录的是意图数据，不只是曝光数据”。

## 验收示例

1. 当用户完成香水分支后，页面显示 `选择：香水`、`路径：perfume_branch`、`停留：45s`。
2. 当只有一次选择事件没有停留时长时，页面仍显示选择结果和路径。
3. 当日志接口失败时，页面展示最近一次本地缓存的选择记录。
