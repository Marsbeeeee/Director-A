# Task 05: 口红分支对比落地

- Status: Planned
- Owner: Product / Frontend / AI
- Last Updated: 2026-04-22
- Source Of Truth: 本文档为 Task 05 当前交付说明
- Depends On: `tasks/task_02_choice_gate.md`
- Related Docs: `demo_flow.md`, `TEMPLATE_task.md`

## 目标

落地“送口红”分支，并与香水分支形成明确对比，让评审看见同一个起点可以走出完全不同的剧情重心。

## 本期范围

1. 定义口红分支的剧情摘要、角色反应和对话重点。
2. 页面上明确展示男嘉宾已选择口红，并进入口红相关剧情。
3. 剧情重点围绕“审美偏好”“自然妆感”“约会风格”展开。
4. 在演示口径上，明确指出它与香水分支的差异。

## 非目标

1. 本期不实现口红色号推荐系统。
2. 本期不加入更多美妆商品选择。
3. 本期不做商品购买链路。

## 关键场景

1. 场景一：用户从同一互动节点选择口红后，进入另一条完整剧情。
2. 场景二：评审能清楚感知这条线强调的是审美偏好，而不是气味氛围。
3. 场景三：现场可与香水分支做前后对比，证明不是同一素材换词。

## 输入 / 输出

### 输入

- 用户输入：来自上一步的 `option_id=lipstick`。
- 上游模块输入：角色设定、口红分支脚本、过渡文案。
- 必填字段：`branch_id`、`branch_title`、`scene_blocks`。
- 可选字段：色号描述、妆感标签、约会风格标签。

### 输出

- 前端展示输出：口红分支剧情内容。
- 后端接口输出：可选返回口红分支结构化内容。
- 持久化输出：记录一次 `branch_entered=lipstick` 事件。

## 状态与异常

- 成功态：口红分支剧情完整展示，且与香水分支差异明确。
- 低置信态：个别剧情块缺失时，用摘要内容兜底。
- 空结果态：口红分支配置为空时，显示默认口红剧情摘要。
- 失败态：加载失败时允许返回节点重选。
- 权限或准入限制：无额外权限限制。

## 数据 / 接口 / UI 约束

- 依赖的数据契约：口红分支至少要有 `setup`、`reaction`、`dialogue_focus` 三段内容。
- 涉及的 API：可复用分支读取接口，或静态配置。
- 是否允许改 schema：允许新增口红分支结构字段。
- 是否允许改接口：允许新增 `branch_id=lipstick` 查询。
- 是否要求兼容现有前端行为：与香水分支使用同一渲染容器。
- 是否需要移动端适配：需要。

## 前端规范补充

- 页面归属：剧情播放器的分支内容区。
- UI 变更点：增加口红分支标题、分支状态标识、剧情块内容。
- 前端消费规则：分支差异由配置提供，前端不应写死对比结论。
- 页面文档同步：如有剧情页文档，应新增口红分支说明。

## 落地范围

- 需要修改的模块：分支配置、剧情渲染区、分支切换逻辑。
- 可以新增的文件类型：口红分支脚本、对比说明文档。
- 本期不应改动的模块：AI 生成卡片基础结构、行为分析面板。
- 测试要求：验证口红分支能独立展示，并与香水分支区分明显。

## 验收

1. 用户点击 `送口红` 后，页面明确进入口红分支。
2. 分支内容必须体现“自然妆感”“审美偏好”“约会风格”。
3. 现场讲解时，能清晰对比口红分支与香水分支的剧情重点差异。
4. 用户能从同一选择节点重复进入两条不同分支。

## 验收示例

1. 当用户选择口红后，页面展示女嘉宾谈到日常妆感和风格偏好。
2. 当口红分支脚本不完整时，系统至少展示摘要版剧情，不出现空白。
3. 当用户从节点重新选择时，系统能正确切到另一条分支。
