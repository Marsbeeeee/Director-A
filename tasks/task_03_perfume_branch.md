# Task 03: 香水分支剧情落地

- Status: Planned
- Owner: Product / Frontend / AI
- Last Updated: 2026-04-22
- Source Of Truth: 本文档为 Task 03 当前交付说明
- Depends On: `tasks/task_02_choice_gate.md`
- Related Docs: `demo_flow.md`, `TEMPLATE_task.md`

## 目标

把“送香水”分支做成一条完整、可信、可讲解的恋综剧情路径，让评审看到品牌商品是如何作为剧情道具进入故事的。

## 本期范围

1. 定义香水分支的剧情摘要、角色反应和对话重点。
2. 页面上明确展示男嘉宾已选择香水，并进入香水相关剧情。
3. 剧情重点围绕“气味偏好”“氛围感”“第一次见面的情绪拉近”展开。
4. 该分支结束后，能够继续衔接到 `Task 04` 的 AI 生成说明。

## 非目标

1. 本期不做真实品牌香水详情页。
2. 本期不做香型推荐算法。
3. 本期不覆盖更多香水子分支。

## 关键场景

1. 场景一：用户在节点选择香水后，剧情立刻切换到香水分支。
2. 场景二：评审观看该分支时，能清楚感知这条线强调的是气味氛围，而不是商品硬卖点。
3. 场景三：分支播放完成后，页面可继续展示“该结果由 AI 生成”的说明层。

## 输入 / 输出

### 输入

- 用户输入：来自上一步的 `option_id=perfume`。
- 上游模块输入：角色设定、香水分支脚本、过渡文案。
- 必填字段：`branch_id`、`branch_title`、`scene_blocks`。
- 可选字段：角色台词、旁白、字幕、商品名称占位。

### 输出

- 前端展示输出：香水分支剧情内容。
- 后端接口输出：可选返回香水分支结构化内容。
- 持久化输出：记录一次 `branch_entered=perfume` 事件。

## 状态与异常

- 成功态：分支内容完整展示，且语义连贯。
- 低置信态：若缺少个别台词，可展示摘要版剧情，不影响主线理解。
- 空结果态：香水分支配置为空时，使用占位剧情摘要。
- 失败态：分支加载失败时，允许返回上一节点重试。
- 权限或准入限制：无额外权限限制。

## 数据 / 接口 / UI 约束

- 依赖的数据契约：香水分支至少要有 `setup`、`reaction`、`dialogue_focus` 三段内容。
- 涉及的 API：可复用剧情分支读取接口，或静态配置。
- 是否允许改 schema：允许新增分支配置字段。
- 是否允许改接口：允许新增 `branch_id` 查询参数。
- 是否要求兼容现有前端行为：分支容器需兼容普通剧情区布局。
- 是否需要移动端适配：需要，关键剧情文本在手机上不能被截断。

## 前端规范补充

- 页面归属：剧情播放器的分支内容区。
- UI 变更点：增加香水分支标题、分支状态标识、剧情块内容。
- 前端消费规则：前端只消费分支结构化字段，不自行拼接剧情含义。
- 页面文档同步：若存在剧情页文档，应新增香水分支说明。

## 落地范围

- 需要修改的模块：分支配置、剧情渲染区、分支状态展示。
- 可以新增的文件类型：分支脚本、占位剧情 JSON、页面文档。
- 本期不应改动的模块：行为分析面板、总结页。
- 测试要求：验证香水分支能独立进入、完整展示并正常返回。

## 验收

1. 用户点击 `送香水` 后，页面明确进入香水分支，而不是只显示“已选择”。
2. 分支内容必须体现“气味偏好”和“氛围感”，不能只展示商品名称。
3. 评审能通过这条分支理解“商品是剧情道具，不是打断内容的广告”。
4. 分支结尾能自然衔接到 AI 生成说明。

## 验收示例

1. 当用户选择香水后，页面展示“女嘉宾闻到香味并主动展开话题”的剧情。
2. 当分支脚本缺少详细台词时，页面仍能用摘要文案讲清剧情推进。
3. 当分支加载失败时，系统允许返回上一步重新选择。
