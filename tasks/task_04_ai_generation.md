# Task 04: AI 分支生成展示

- Status: Planned
- Owner: Product / AI / Frontend
- Last Updated: 2026-04-22
- Source Of Truth: 本文档为 Task 04 当前交付说明
- Depends On: `tasks/task_03_perfume_branch.md`
- Related Docs: `demo_flow.md`, `TEMPLATE_task.md`

## 目标

把“AI 生成后续内容”这件事展示清楚，让评审知道当前剧情不是预写死脚本，而是根据用户选择动态延展出来的。

## 本期范围

1. 在香水分支后增加一块 AI 生成说明区或生成结果卡片。
2. 明确展示本次生成的输入依据：当前选择、用户偏好标签、剧情目标。
3. 明确展示输出结果：后续剧情方向、情绪基调、对话重点。
4. 保证这块内容适合现场讲解，不要求展示底层模型细节。

## 非目标

1. 本期不展示完整 Prompt。
2. 本期不接入复杂模型评估体系。
3. 本期不要求生成长篇完整剧本。

## 关键场景

1. 场景一：评审在香水分支结束后，看见“当前剧情由 AI 按选择结果生成”的说明。
2. 场景二：若实时生成较慢，可展示预生成结果，但说明结构必须一致。
3. 场景三：生成失败时，系统回退到默认分支文案，不阻断 Demo。

## 输入 / 输出

### 输入

- 用户输入：无新增直接输入，继承上一步选择结果。
- 上游模块输入：`option_id`、用户偏好标签、当前剧情上下文。
- 必填字段：`selected_option`、`user_profile_tags`、`generation_goal`。
- 可选字段：风格偏好、品牌关键词、输出长度。

### 输出

- 前端展示输出：AI 生成说明卡、后续剧情摘要。
- 后端接口输出：可新增 `POST /ai/generate-story`。
- 持久化输出：记录一次 `ai_generation_shown` 事件及生成摘要。

## 状态与异常

- 成功态：页面展示输入依据和生成结果，逻辑可讲清。
- 低置信态：若模型返回较弱，可用预设结果兜底并标记为示例结果。
- 空结果态：生成返回空时，回退为默认剧情摘要。
- 失败态：生成失败不应中断 Demo，要允许继续播放分支内容。
- 权限或准入限制：如依赖 API Key，仅限内部演示环境。

## 数据 / 接口 / UI 约束

- 依赖的数据契约：生成结果至少包含 `inputs`、`output_summary`、`tone`。
- 涉及的 API：`POST /ai/generate-story` 或等价 mock。
- 是否允许改 schema：允许新增 AI 结果展示结构。
- 是否允许改接口：允许新增生成接口或 mock 返回。
- 是否要求兼容现有前端行为：无 AI 返回时也必须可展示默认内容。
- 是否需要移动端适配：需要，生成卡片不能遮挡主剧情。

## 前端规范补充

- 页面归属：香水分支后的说明层或侧边信息区。
- UI 变更点：增加输入依据、标签、生成结果摘要区域。
- 前端消费规则：前端只负责展示结构化生成结果，不拼底层策略解释。
- 页面文档同步：如有 AI 展示页说明，应补充该卡片结构。

## 落地范围

- 需要修改的模块：AI 结果卡片、分支页面衔接、生成接口或 mock。
- 可以新增的文件类型：mock 数据、生成结果模板、页面文档。
- 本期不应改动的模块：口红分支、行为数据统计逻辑。
- 测试要求：验证成功、空结果、失败回退三种状态。

## 验收

1. 页面必须能清楚说明“当前后续剧情由 AI 根据选择生成”。
2. 展示结果里必须包含至少一项输入依据和一项输出摘要。
3. 即使 AI 生成失败，Demo 也能继续讲下去。
4. 评审能听懂 AI 在这条链路里承担的角色，而不是只看到一个标签。

## 验收示例

1. 当用户选择香水时，页面展示“输入：香水、偏浪漫、重氛围感；输出：加强气味记忆与情绪拉近”。
2. 当生成接口超时时，系统展示预生成的默认结果卡，并允许继续演示。
3. 当生成结果为空时，页面回退到默认剧情摘要，不出现空白区。
